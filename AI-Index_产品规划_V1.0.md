# AI Index —— AI智能指数基金投资平台（产品规划 V1.0）

> **定位：** AI 驱动的指数基金 **投资决策** 平台，而不是交易软件。  
> **硬边界：** 不做实盘下单、不做券商 API、不代客执行买卖；输出仅为可解释建议与提醒，由用户自行在外部渠道交易。

---

# 1. 产品愿景

打造一款帮助长期投资者完成 **指数筛选、估值分析、趋势分析、仓位管理、风险控制、智能提醒** 的 AI 投资助手。

## 核心价值

- 一键筛选指数基金
- AI 趋势分析（概率而非确定性预测）
- 智能仓位建议（含成本感知后的净建议）
- AI 投资教练
- 自动提醒与再平衡建议
- 策略市场（研究与分享规则，非跟单执行）

## 明确不做

| 不做 | 说明 |
|------|------|
| 实盘下单 / 券商对接 | 不连接交易通道，不下单、不成交、不持仓同步券商 |
| 自动跟单 / 程序化执行 | 提醒与建议停留在「决策层」，执行由用户人工完成 |
| 资金托管 / 账户权限 | 本地配置与缓存，不接触用户交易密码与资金 |

## 智能仓位

根据：

- 用户资产
- 风险偏好
- 现金比例
- 目标配置
- **交易成本假设**（见 §3.1，仅用于建议与回测，不触发下单）

自动计算：

- 建议买什么 / 买多少（金额与比例）
- 是否建议减仓
- 是否建议再平衡
- **扣成本后的净偏离是否值得动作**（低于成本阈值则提示「观望」）

> 以上均为决策输出；用户需自行在银行/券商/基金 App 完成交易。

## 3.1 交易成本（规划纳入，不执行交易）

目标：让仓位建议、再平衡提醒与策略回测 **看见成本**，避免「纸面最优、实盘磨损」。

| 成本项 | 用途 | 规划口径 |
|--------|------|----------|
| 申购费 / 赎回费 | 场外基金建议 | 可配置费率表；同一基金公司转换可单独建模 |
| ETF 佣金 + 滑点 | 场内 ETF 建议 | 佣金（bps）+ 可选冲击成本（按成交额分档） |
| 管理费 / 托管费 | 长期持有对比 | 年化费率折入持有成本，用于筛选与解释 |
| 再平衡阈值门槛 | 提醒降噪 | 「建议变动金额 × 预估费率」低于阈值则不发待执行提醒 |

落地原则：

1. 成本只进入 **建议金额、提醒分级、回测净收益**，不进入任何下单链路。  
2. 默认给出保守假设，允许用户在配置中覆盖。  
3. V2 回测报告必须同时展示 **毛收益 / 净收益（扣成本）**。

## 3.2 数据信号（规划补齐）

目标：把「可计算、可解释、可缓存」的公开数据信号写进评分与趋势引擎；信号用于决策，不用于自动交易。

### 已具备（当前实现基线）

- 指数 OHLCV 日线 → 趋势分、偏多概率、多展望周期  
- 指数 PE/PB 百分位 → 估值档位与仓位系数  
- 宏观：PMI / CPI / M2 / LPR → 宏观与政策因子  
- 规则化八因子与 Coach 解释  

### 规划补齐的数据信号

| 信号 | 用途 | 优先级 |
|------|------|--------|
| ETF / 指数基金成交额 | 流动性、冲击成本代理、机会过滤 | P0 |
| 跟踪误差 | 选基与「指数暴露纯度」 | P0 |
| 申赎 / 份额变动（可得范围内） | 资金因子增强 | P1 |
| 指数盈利增速 / 一致预期（公开源） | 盈利因子增强 | P1 |
| 行业/主题拥挤度（成交额占比、估值分位） | 风险与情绪 | P1 |
| 波动率体制（已有 ADX/ATR，可扩展 IV 代理） | 趋势门控与仓位上限 | P1 |
| 北向 / 主力资金（公开接口可得时） | 资金与情绪 | P2 |
| 费率与规模时序 | 筛选与持有成本 | P2 |

信号治理要求：

- 每条信号标注 **来源、更新频率、缺失降级策略**；  
- 写入评分时保持 **可解释权重**，避免黑箱；  
- 样本外校准优先覆盖趋势与关键因子权重（Walk-Forward），成本模型与信号一并进入净绩效评估。

# 5. 回测实现规划（决策层，不执行交易）

> 现状：`jijin/engine/calibrate.py` 只做趋势参数 Walk-Forward **选参**，不是完整策略回测。  
> 目标：补齐可解释、可配置、含成本的回测报告；仍不下单。

## 5.1 目标与非目标

| 做 | 不做 |
|----|------|
| 用历史指数日线验证「信号 → 仓位规则 → 建议动作」是否站得住 | 券商撮合、实盘、滑点撮合引擎 |
| 毛收益 / 净收益（扣成本）对照 | 高频、日内、杠杆融资 |
| Walk-Forward / 样本外报告，降低过拟合叙事 | 黑箱 ML 全自动调参取代规则解释 |
| 与现有趋势引擎、仓位系数、成本配置共用口径 | 引入重型运行时依赖（如 vectorbt 仅作研究对照，不进主路径） |

## 5.2 分层：三类回测由浅入深

```text
L1 信号回测（Signal）     ← 扩展现有 calibrate
L2 规则组合回测（Portfolio Rules）
L3 决策链回测（Decision Chain，对齐产品建议）
         │
    成本模型 CostModel（§3.1）贯穿 L2/L3
```

### L1 — 信号回测（优先，复用校准）

验证趋势分 / 偏多概率对未来收益的预测力。

- **输入**：指数池、展望周期 `forward_days`、当前或候选 `trend` 参数  
- **持仓代理**：`score≥阈值` 多头，否则空仓（与校准一致，可配置阈值）  
- **指标**：命中率、IC、Sharpe、胜率、最大回撤（日度权益曲线）  
- **样本外**：沿用 Walk-Forward（train/test/step）；全样本仅作对照，不单独作为「可接受」依据  
- **产出**：净值曲线、分折指标表、与默认参数对比  

实现要点：从 `calibrate._evaluate_window` 抽出 `BacktestMetrics` + 权益序列，校准与回测共用评估核。

### L2 — 规则组合回测

验证「估值档位 × 计划仓位」类再平衡规则（对齐智能仓位，不要求八因子齐全）。

- **输入**：`portfolio.index_plans`、估值百分位→仓位系数表、`min_rebalance_pct`、成本假设  
- **模拟**：按日/周调仓；仅当 |Δ仓位| 超过阈值且 **扣成本后仍有净改善** 才调仓  
- **基准**：等权持有计划指数、或长期固定目标仓位  
- **指标**：年化、回撤、换手、调仓次数、毛/净收益差  

数据依赖：历史 PE/PB 百分位序列（乐咕等，已有估值源）；缺失日用前值填充并标注覆盖率。

### L3 — 决策链回测（完整产品口径）

近似复现「重点机会 + 仓位建议」在历史上会怎么做（研究用，接受简化）。

- **信号**：默认展望周期的趋势偏多概率（或八因子总分，可开关）  
- **选池**：每日/每周取 Top-K 机会（或固定观察池）  
- **仓位**：风险偏好模板 × 估值系数 × 成本阈值  
- **报告**：相对基准超额、换手、成本拖累、分年度绩效  
- **门控**：须先有 L1/L2 稳定，再开放 L3，避免一次做大

## 5.3 模块与配置设计

建议目录（新建，不把回测塞进 `calibrate.py`）：

```text
jijin/engine/cost.py           # CostModel：费率、滑点 bps、再平衡成本阈值
jijin/engine/backtest/
  __init__.py
  types.py                     # BacktestRequest / EquityCurve / Report
  metrics.py                   # Sharpe、回撤、年化、换手、分年度
  signal.py                    # L1：信号→仓位代理→权益
  portfolio_rules.py           # L2：估值仓位规则
  decision_chain.py            # L3：机会+规则（后置）
  walk_forward.py              # 滚动窗编排（供校准与回测共用）
jijin/agents/__init__.py       # backtest_agent(...)
app.py                         # 新页或「策略参数」下「回测报告」Tab
tests/test_backtest_*.py
```

配置草案（`config.yaml`）：

```yaml
costs:
  etf_commission_bps: 3          # 单边
  etf_slippage_bps: 5
  fund_subscribe_rate: 0.0015    # 场外申购，可按基金覆盖
  fund_redeem_rate: 0.005
  management_fee_annual: 0.005   # 缺省；持有成本日摊
  min_trade_edge_bps: 15         # 低于此「成本阈值」不调仓

backtest:
  years: 5
  mode: signal                   # signal | portfolio_rules | decision_chain
  rebalance: weekly              # daily | weekly | monthly
  forward_days: 21
  top_k: 5
  benchmark: equal_weight_plans
  walk_forward:
    train_days: 504
    test_days: 63
    step_days: 63
```

## 5.4 成本如何进入回测（与 §3.1 对齐）

1. **调仓日**：对买入/卖出名义本金分别扣佣金+滑点（或申赎费）。  
2. **持有期**：管理费按日从权益中扣除（可选开关）。  
3. **报告**：始终双列 **Gross / Net**；净收益为接受策略的主指标。  
4. **与提醒共用**：同一 `CostModel` 供智能仓位「是否值得再平衡」使用，避免两套费率。

## 5.5 界面与 CLI

| 入口 | 内容 |
|------|------|
| Streamlit「回测报告」页（或策略参数 Tab） | 模式选择、指数池、年限、成本开关；权益曲线；毛/净指标卡；分年度表；Walk-Forward 折叠区 |
| CLI `python -m jijin backtest --mode signal` | 便于回归与无 UI 环境 |
| 导出 | CSV / Markdown 摘要（可选） |

展示原则：红涨绿跌与主站一致；注明「模拟研究，非实盘」。

## 5.6 与自动校准的关系

| | 自动校准 | 回测报告 |
|--|----------|----------|
| 目的 | 搜索更优趋势参数 | 评估既定规则历史表现 |
| 输出 | 建议写回 `trend` | 报告，默认不改配置 |
| 共用 | 行情加载、Walk-Forward 窗、信号评估核、成本模型 | 同左 |

校准成功后可一键「用当前参数跑 L1 回测」；回测不自动写配置。

## 5.7 分阶段交付

| 阶段 | 版本对齐 | 交付物 | 验收 |
|------|----------|--------|------|
| **B0** | V1.5 前半 | `CostModel` + 配置；仓位建议侧可读费率（未必先改提醒） | 单测覆盖费率计算 |

| **B3** | V2.0 | L3 决策链简化回测 + 分年度 + Walk-Forward 汇总报告 | 文档示例跑通；标明简化假设 |
| **B4** | V2.0+ | 八因子/机会 Top-K 纳入 L3；导出报告 | 信号缺失时有降级说明 |

## 5.8 风险与简化假设（必须写在报告页）

- 用**指数点位**代理可交易 ETF/基金净值，存在跟踪误差（待 §3.2 跟踪误差信号补齐后可修正）。  
- 忽略停牌、涨跌停、额度与流动性枯竭。  
- 估值/宏观历史若缺失，前值填充会高估稳定性。  
- L3 无法逐 bar 复现 UI 全链路并发与缓存，只保证规则口径一致。

## 5.9 测试策略

- 夹具：固定短行情 CSV → 权益与回撤金标。  
- 成本：零费率时 Net==Gross；提高佣金后 Net 严格下降。  
- Walk-Forward：窗边界无未来函数（调仓用 `shift(1)` 信号）。  
- 不强制 CI 跑联网 5 年全量。
